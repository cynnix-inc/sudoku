Rule: Commit Style (Conventional Commits)
Applies to: all commits
Use when: writing commit messages
Avoid: vague messages like "update" or "fix stuff"
Definition of Done:
  - Message follows <type>(scope): summary
  - Body explains what/why if non-trivial

Types: feat, fix, chore, docs, refactor, test, build, ci, perf
Examples:
- feat(game): add candidate highlighting
- fix(ui): correct status bar color on Android
- chore(ops): add CI and PR template
