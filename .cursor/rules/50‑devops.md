# DevOps, CI, and Releases

Tooling
- Node 22 LTS pinned. pnpm package manager.
- Husky + lint‑staged, commitlint with Conventional Commits.

CI (GitHub Actions)
- On PR: install with cache, typecheck, lint, test, build web, upload coverage.
- Optional label `automerge` enables GitHub Auto‑merge once checks are green.
- Branch protection: require checks and conversation resolution.

Releases
- Trunk‑based on `main`. Work in short‑lived branches.
- EAS channels: development, preview, production.
- Use preview updates on PRs. Promote to production from `main`.
- Versioning and changelog handled by Changesets.

Docs
- README has autogenerated sections for scripts and EAS profiles.
- When env or channel changes, update `.env.example`, `eas.json`, README, and `docs/`.

Local on Windows
- Use Volta or nvm‑windows to pin Node. Enable Developer Mode for symlinks if needed.
